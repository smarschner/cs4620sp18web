WebGL

  • According to the WebGL's official website:
    WebGL is a cross-platform, royalty-free web standard for a low-level 3D graphics API based on OpenGL ES 2.0 [...]
  • Some interesting things to note:
    • Cross platform. It runs on Windows, Mac, Linux, Android, iOS, etc.
    • Web. It is for writing web applications; in other words, programs that run in web browsers. There are also a lot of browsers supporting it. You can find out which browsers support WebGL in this web page.
    • Low level. It talks to the GPU to do its job.
    • API. It's a library. It provides commands that you are supposed to compose to do the job you want done yourself.
  • OpenGL
    • Stands for "Open Graphics Library."
    • A low-level API for 2D/3D rendering with the GPU.
    • It is cross-platform.
    • Main atlernative: DirectX, Vulkan
  • OpenGL ES
    • OpenGL for Embedded Systems
    • Subset of OpenGL API for use with smartphones, tablets, and video game consoles.
    • Game oriented.
    • History
      • 2003: OpenGL ES 1.0 (Android, Sybian, Playstation 3)
      • 2007: OpenGL ES 2.0 (Android 2.0, iOS 5, WebGL, etc)
      • 2012: OpenGL ES 3.0 (Android 4.3, iOS 7, WebGL 2, etc)
  • Programming Environment
    • Use Javascript to program in the browser.
    • Use OpenGL Shading Language (GLSL) to program shaders.
  • Very useful resource: WebGL 1.0 Reference Card.

The Graphics Pipeline

  • An algorithm for rendering 3D geometry that is implemented in GPUs.
  • Two approaches to rendering:
    • Object order or rasterization 
      for each object in the scene {
    for each pixel in the image {
        if (object affects pixel) {
            do something
        }
    }
}
    • Ray order or ray tracing 
      for each pixel in the image{
     for each object in the scene {
        if (object affects pixel) {
            do something
        }
    }
}
  • The graphics pipeline is an instance object order rendering.
  • Input & output
    • Input: Vertices and connections among them + other types of data (texture, etc.)
    • Output: Pixels on the framebuffer.
  • Some jargons:
    • Buffer = "a region of physical memory used to temporarily store data " (according Wikipedia).
      • In our context, a buffer mostly resides in the GPU.
    • Framebuffer = a buffer that holds the image that is displyaed on the monitor.

  • Some explanation:
    • Vertex = a point. Can have attributes.
      • Position
      • Texture coordinates
      • Normal vector
      • etc.
    • Primitive = a geometric shape specified by a collection of vertex.
      • Point = 1 vertex
      • Line = 2 vertices
      • Triangle = 3 vertices (This is the most important primitive.)
      • Quad = 4 vertices
      This is what we mean by "vertex connectivity."
    • Rasterization = the process of turning vertex+primitive data small pieces, each of size equal to a pixel.
    • Fragment = a potential pixel, resulted from rasterization.
    • Raster operations = processing fragments+framebuffer to decide what gets written to the framebuffer
      • Depth test
      • Blending
      • Stencil test
      • etc.
    • A vertex shader transform vertices into clip space.
      • We will learn about clip space later in CS 4620.
    • A fragment shader process info about fragments and output a color or decide to discard the fragment.
    • A "program" is a collection of shaders. In WebGL, you must have a vertex shader and a fragment shader.
